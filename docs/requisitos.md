# 3. Requisitos do Sistema

## 3.1 Requisitos Funcionais (RF)

RF01 — O sistema deve permitir o cadastro, edição e remoção de alunos.  
RF02 — O sistema deve registrar pagamentos e atualizar o status do aluno (ativo/inativo).  
RF03 — O sistema deve permitir ao aluno reservar uma vaga em aula coletiva.  
RF04 — O sistema deve impedir reservas quando as vagas estiverem esgotadas.  
RF05 — O aluno deve poder cancelar uma reserva de aula.  
RF06 — O instrutor deve poder registrar e atualizar treinos personalizados.  
RF07 — O administrador deve poder gerenciar planos da academia.  
RF08 — O sistema deve permitir o controle de entrada e saída de alunos ativos.  
RF09 — O sistema deve gerar relatórios (frequência, status financeiro, reservas).  
RF10 — O aluno deve poder visualizar seu histórico de pagamentos.  

---

## 3.2 Requisitos Não Funcionais (RNF)

RNF01 — O sistema deve ser responsivo e acessível em dispositivos móveis.  
RNF02 — O sistema deve garantir que apenas alunos ativos possam entrar na academia.  
RNF03 — O tempo máximo de resposta deve ser inferior a 2 segundos.  
RNF04 — O sistema deve armazenar dados com segurança e integridade.  
RNF05 — A interface deve ser intuitiva e fácil de usar.  
RNF06 — O sistema deve operar 24/7 com disponibilidade mínima de 95%.  
RNF07 — Dados sensíveis devem ser criptografados.
